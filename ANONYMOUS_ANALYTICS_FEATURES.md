# Anonymous Analytics Features - NHS Compliance

## Overview
This implementation adds two core success measurement features while maintaining **ZERO Personal Data Collection** policy and full NHS/GDPR compliance.

---

## Feature 1: Anonymous Monthly Caregiver Burden Survey

### Objective
Measure changes in caregiver stress using a validated scale (ZBI/GAD-7) without linking any PII.

### Implementation

#### Database Schema
- **Table**: `survey_responses`
- **Stored Data** (ONLY):
  - `code_hash` - 17-digit anonymous code (hashed)
  - `completion_date` - Date survey was completed
  - `result_bucket` - 'Low', 'Medium', or 'High' burden
  - `survey_day` - Milestone day (30, 60, 90, etc.)
- **NOT Stored**: No text answers, no PII, no IP addresses

#### Trigger Logic
- Survey eligibility checked on dashboard load
- Account age calculated from `created_at` timestamp
- Banner appears at Day 30, 60, 90, 120, etc.
- Each milestone can only be completed once
- Banner can be dismissed (cached for 24 hours)

#### Survey Flow
1. **Eligibility Check**: `GET /api/survey/check-eligibility/:code_hash`
2. **Banner Display**: Purple gradient banner at top of caregiver dashboard
3. **External Survey**: Opens Tally form (GDPR/HIPAA compliant)
4. **Result Recording**: `POST /api/survey/record-completion`
5. **Aggregated Stats**: `GET /api/survey/aggregate-stats`

#### Privacy Safeguards
✅ 17-digit code passed as encrypted URL parameter  
✅ External survey tool configured with NO IP logging  
✅ Only result bucket stored (Low/Medium/High)  
✅ No individual responses retained  
✅ Statistical aggregation only

## Feature 2: Cumulative Daily Active User (DAU) Tracking

### Objective
Track app engagement (the "3 AM Test") using aggregated metrics to prove consistent daily usage.

### Implementation

#### Database Schema
- **Permanent Table**: `daily_active_users`
  - `event_date` - Date of launches
  - `event_hour` - Hour of day (0-23)
  - `launch_count` - Aggregated count
  - **NO individual codes stored**

- **Temporary Table**: `daily_launch_tracker` (auto-cleaned after 48 hours)
  - Used only to prevent double-counting same code on same day
  - Automatically purged every 2 days

#### Tracking Flow
1. **App Launch**: Triggered on page load (logged-in users only)
2. **Uniqueness Check**: Check if code already launched today
3. **Increment Aggregate**: Add +1 to hourly count
4. **Discard Code**: Code is immediately discarded from permanent storage
5. **Auto-Cleanup**: Old tracker records deleted automatically

#### Privacy Safeguards
✅ Only ONE launch per code per 24-hour period  
✅ Code discarded immediately after incrementing count  
✅ Only aggregated hourly totals stored permanently  
✅ Temporary tracker auto-cleaned every 48 hours  
✅ No user identity linkable to launch events  

#### Public Disclosure Wording
```
"Cumulative Daily Active Users are tracked by counting the total number 
of unique, daily 'App Launch' events generated by anonymous codes. 
We do not track the identity of the user."
```

## Compliance Checklist

### NHS Digital Standards
- ✅ No PII collected or stored
- ✅ Anonymous identifiers only (hashed codes)
- ✅ Aggregated statistics only
- ✅ Data minimization principle
- ✅ Automatic data retention limits
- ✅ Purpose limitation (analytics only)

### GDPR Article 25 (Data Protection by Design)
- ✅ Anonymization at source
- ✅ Minimal data retention
- ✅ No re-identification possible
- ✅ Transparent processing (public wording)
- ✅ Limited purpose specification

### Key Metrics Tracked
1. **Survey Success**: Mean reduction in caregiver burden across cohorts
2. **App Engagement**: Daily active users, hourly distribution
3. **Retention**: Account age vs survey participation
4. **Usage Patterns**: Peak usage hours (for optimization)

## Maintenance

### Auto-Cleanup Jobs
The following runs automatically on every app launch request:
```sql
DELETE FROM daily_launch_tracker 
WHERE launch_date < CURRENT_DATE - INTERVAL '2 days';
```

### Manual Cleanup (if needed)
```sql
-- Remove all tracker data older than 2 days
DELETE FROM daily_launch_tracker 
WHERE launch_date < CURRENT_DATE - INTERVAL '2 days';

-- View current DAU aggregates
SELECT event_date, SUM(launch_count) as total
FROM daily_active_users
WHERE event_date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY event_date
ORDER BY event_date DESC;
```

## Ethical Considerations

### Why This Approach?
- **Patient Privacy First**: Zero risk of re-identification
- **NHS Trust**: Maintains strict data protection standards
- **Regulatory Compliance**: Exceeds minimum GDPR requirements
- **Research Validity**: Aggregated data still scientifically valid
- **Transparency**: Clear public disclosure of tracking methods

### What We DON'T Track
❌ User names  
❌ IP addresses  
❌ Device IDs  
❌ Location data  
❌ Session duration  
❌ Individual survey text responses  
❌ Behavioral patterns linked to individuals  

### What We DO Track
✅ Anonymous code participation (hashed)  
✅ Survey completion milestones  
✅ Result buckets (Low/Medium/High)  
✅ Daily launch counts (aggregated)  
✅ Hourly usage patterns (aggregated)
